# OCSS Command Center - 2-Minute Emergency Demo Script

**Purpose:** Use this when meeting time is cut and you need to show core value fast.  
**Duration:** ~2 minutes total  
**Format:** Verbatim script + exact click path

---

## 0:00-0:20 — Opening Line

“Given time, I’ll show three screens that prove end-to-end control: ingestion safety, row-level case execution, and leadership KPI visibility.”

---

## 0:20-0:55 — Screen 1: Program Officer Ingestion

**Click path**
1. Sidebar → **Program Officer**
2. Open **📤 Upload & Processing**
3. Point to metadata fields and processing controls

**Say**
“Reports are ingested with metadata, assigned by caseload, and protected by duplicate-period checks. Each successful ingestion gets a confirmation ID, which improves auditability and operational trust.”

---

## 0:55-1:35 — Screen 2: Support Officer Row-Level Work

**Click path**
1. Sidebar → **Support Officer**
2. Open **📝 My Assigned Reports**
3. Select a report queue item and a single case row

**Say**
“Each line in a report is treated as a case. Support Officers process one row at a time, save per-row status, and the system blocks supervisor submission until assigned rows are completed. This is how we handle 50+ line reports safely.”

---

## 1:35-2:00 — Screen 3: Director KPI/Ticket Oversight

**Click path**
1. Sidebar → **Director**
2. Open **🆘 Ticket KPIs**
3. Point to filters and KPI metrics

**Say**
“Leadership gets real-time visibility of both production workflow and support health. KPIs reflect actual row-level work, and ticket analytics can be filtered by scope, category, priority, and time range.”

---

## 10-Second Close

“This gives us controlled intake, accountable execution, and leadership visibility in one operating workflow — ready for pilot and governance hardening next.”
