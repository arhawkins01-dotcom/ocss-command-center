Command Center Report Processing

This document describes the unified operational workflow for processing Establishment Command Center reports (56RA, Locate, P-S).

Purpose

Replace spreadsheet-based workflows with a centralized Command Center that ingests report datasets, assigns work, tracks status, and records case narration.

Core Processing Model

1. Review Assigned Case
2. Determine Next Action
3. Process Required Activity
4. Update Action Status
5. Complete Case Narration

Universal Processing Steps

- Open Assigned Work Queue: Dashboard → Assigned Reports → Select Case
- Review Case Status: SFRC, PEGT, CTHS, ILSU, Court Dockets, Locate Databases
- Process Required Action: Schedule GT, Prep ADS, Refer to Court, Issue Contact Letter/COBO, Postal Verification, Complete Locate Efforts, Close Case, File in OnBase
- Update Command Center Fields: Date Action Taken, Action Taken / Status, Case Narrated (Y/N), Comments
- Case Narration Standard: Status of Case; Systems Reviewed; Action Completed; Next Steps

Example Narration

Command Center Review: Status verified in SFRC.
Locate sources checked. Postal verification issued.
Case pending next SOP action.

Supervisor Monitoring

Supervisors use dashboards for completion metrics and analytics. Manual email and spreadsheet tracking are deprecated.
